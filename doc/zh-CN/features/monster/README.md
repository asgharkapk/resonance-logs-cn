# 怪物监控

对应工具箱中的 **怪物监控**，在游戏内单独浮窗展示**当前攻击主目标**身上的 Buff、队友相关 Buff 与仇恨列表。

与 **实时监控**（自身技能 CD、自身 Buff 等）相互独立：配置保存在「怪物监控」内，使用 **怪物遮罩** 窗口显示，不会混用实时监控的方案。

## 概述

- **当前攻击主目标**：浮窗中的**怪物 Buff** 与**仇恨列表**会跟随你角色当前的**攻击目标**自动切换（读游戏内目标 ID），无需再配置 `ExtraBuffMonitoredMonsters.json`（该配置已废弃）。
- **切换时机**：选中并攻击另一只怪物时，显示会切到该目标；目标死亡或丢失选中后，对应 Buff / 仇恨会清空，直到有新的主目标。
- **木桩 / 小怪**：只要该单位能成为你的攻击目标并被游戏同步到，即可像 Boss 一样显示；须**以木桩为当前攻击目标**（对其造成伤害或选中），而不是只看附近有没有木桩。

> 若 Buff 名称难辨认，可在各 Tab 的 **显示名称** 中为已监控条目设置别名。

## 启用与顶栏操作

- **启用怪物监控**：页面顶部总开关；关闭后不再向怪物遮罩推送数据。
- **切换怪物遮罩**：显示或隐藏游戏内怪物监控浮窗。
- **编辑怪物布局**：在浮窗上拖动、缩放各区域（怪物 Buff 区、队友 Buff 区、仇恨列表区、Boss DBM 区等）。

怪物监控支持多个**配置子方案**。当前总方案会引用其中一个；切换总方案，或在怪物监控页面切换子方案，即可一并切换 Buff、仇恨、DBM、语音播报和布局等配置。顶部总开关、日常场景自动隐藏和 Buff 别名仍为全局设置，不随方案切换。

## 与实时监控的区别

| 项目 | 实时监控 | 怪物监控 |
|------|----------|----------|
| 浮窗 | 游戏遮罩（game-overlay） | 怪物遮罩（monster-overlay） |
| 主要对象 | 自身技能、自身 Buff、属性等 | 当前主目标 Buff、队友 Buff、仇恨 |
| 方案 | 多套配置子方案，可由总方案引用和切换 | 多套配置子方案，可由总方案引用和切换 |
| Buff 范围 | 玩家自身相关 | 怪物身上 + 队友施加到怪物 |
